# SSO Routes And Roads

Keywords: sso route routes roads avalonian roads portal scout scouting ttl close closes path chain deep road bridgewatch martlock lymhurst fort sterling thetford caerleon brecilien

SSO routes and road scouting should stay organized on the route board.

The SSO route board is a single clean display message. Scouts should use the
board's Add / Update Route button instead of typing loose route messages in the
channel. When a scout submits a route, the board shows it immediately as a
pending route so members can use the chain. Officers still approve the bounty
payout afterward.

Good route post format:

- Starting point
- Portal chain or zone chain
- Destination
- Scout name
- TTL / close time if known
- Notes such as danger, hostile groups, or useful objectives

Guidance:

- If a route changes, update the route board instead of posting a messy new
  message.
- If a route is awaiting approval, it can still be used, but the scout's bounty
  payout is not final until an officer approves it.
- If a route is closed, mark it closed when possible.
- Routes are not the same as LFG events. Do not tell members to use the SSO
  route board when they ask how to create or join LFG.
- Roads content should include extraction plan and clear regroup points.
