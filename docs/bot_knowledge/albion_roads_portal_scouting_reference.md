# Albion Roads Portal And SSO Scouting Reference

Keywords: roads avalon avalonian sso route portal portals charges ttl lifetime close close time scout scouting hideout ho current route add update mark closed chain connection restock

Use this for SSO route questions, Roads portal mechanics, and route board help.

Roads basics:

- Roads of Avalon are dynamic portal networks that connect zones across Albion.
- Portal routes change, close, and re-roll. A route that was safe earlier may be
  dead or camped later.
- Portals have limited charges and restock over time. Solo/group/raid style
  portal types have different charge behavior, so a large group can burn a route.
- Tunnel lifetimes are limited, and route scouts should report TTL/close time
  when known.

SSO route board policy:

- Use the SSO route board's Add / Update Route button when possible.
- Submitted routes show on the route board immediately as pending approval so
  members can use the route.
- Officer approval is still required for the bounty payout.
- Mark the route closed when the chain is dead, camped beyond use, or the timer
  is wrong.

Good SSO route submission:

- Start from the hideout or known origin.
- List portal chain in order.
- Include destination or useful exit.
- Include TTL, for example `ttl: 1h30m`, `ttl: 90m`, `ttl: 12hr 52min`.
- Add note: danger, gankers, distance from portal, faction pressure, mammoth
  safety, or useful objectives.

Example:

`SSO > Bleachskull Desert > note: 2 zones from Martlock portal > ttl: 12h`

Answer style:

- "Post it with the board button so it updates the single clean route board."
- "If you typed it manually, an officer may need to move it into the board or
  ask you to use Add / Update Route."
- "Do not treat a Roads chain as safe just because it exists. Scout it."
